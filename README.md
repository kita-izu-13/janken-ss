## 概要<br>
・SS学習開始約1か月後にEclipseで作成したプログラムです。<br>
　学習用プログラムをもとに、改変しました。<br>
・理解を深める目的として作成した、<br>
　Web上で動作するじゃんけんゲームです。<br>
・Java（servlet、model）とJSPを中心に、<br>
　HTML / CSS / JavaScript を組み合わせ、<br>
　画面操作とサーバ側処理の連携を意識して実装しました。<br>
　（HTML＆CSS学習約1カ月、JavaScript学習1カ月）<br>
　　　> [▲ トップへ戻る](#top)

## できる事<br>
・ コンピュータとのじゃんけん<br>
1. ログイン（簡易的なID・パスワード認証）<br>
1. じゃんけん選択時、カーソル変更<br>
1. グー、チョキ、パーのボタン選択で勝負。<br>
1. 勝敗結果を最新順で画面に表示<br>
1. ログアウト時、確認ダイアログ表示<br>

## 工夫した点<br>
・選択時に視覚的に分かるよう、<br>
　JavaScriptでマウスオーバー時に「カーソルを画像に」変更。<br>
　JavaScriptで処理を実装することで、<br>
　ID取得やカーソルが重なった時・外れた時のスタイル変更処理の<br>
　理解を深めることを意識しました。<br>

・じゃんけん処理については、制作効率を考え、　<br>
 「コマンドプロンプト上で動くじゃんけんプログラム」をもとにしました。<br> 

・当初はラジオボタンで選択して「対戦！」としてましたが、<br>
　即座に選択して結果が出るように、ボタン送信（submit）に変更しました。<br>
　servletでは、name/valueを用いて選択内容を判定。<br>

・CSSについては、検証として<br>
　静的インクルードで取り込みました。<br>

・一つずつ機能を実装しながら、<br>
　ユーザー視点も考えつつ、改善・試行錯誤しました。<br>

## 苦労や学び<br>
・実装していくにつれて、「こうだったらいいな」を形にするため、<br>
　次々と機能を増やしていきました。<br>
　その時点で学習しているものを活かせるように、<br>
　アウトプット用の作品として制作しました。<br>

・苦労して実装した機能の後は、<br>
　別で学習した、比較的すぐにできる機能を実装したり、見た目を変える事で、<br>
　モチベーションの維持に努めました。<br>
 
・JavaScriptについて、<br>
　保存場所やパスをどうしたらいいのかが分からず悩みました。<br>
　そのおかげで、調べて実際にどうしたらいいのかが分かり、<br>
　実体験として経験になったので良かったと思います。<br>

・エラーが多く発生したり、イメージとの相違はありましたが、<br>
　一つずつ原因を確認・修正することで、<br>
　最終的にイメージ通りに動作させることができました。<br>
